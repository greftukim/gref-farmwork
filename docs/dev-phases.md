# 개발 단계

## 1단계: 프로젝트 세팅 + 인증

1. Vite + React + Tailwind 프로젝트 생성
2. PWA 설정 (vite-plugin-pwa)
3. 프로젝트 디렉토리 구조 생성
4. 공용 컴포넌트 (Button, Card, Modal, BottomSheet, TopBar, BottomNav)
5. 로그인 페이지 (PIN 로그인, 목업 계정)
6. 관리자/작업자 레이아웃 분기 (AdminLayout, WorkerLayout)
7. React Router 라우팅 설정
8. 개발용 목업 데이터 (Zustand 스토어에 시드)

## 2단계: 인사 관리 (HR)

1. 직원 관리 페이지 — 목록 테이블, 등록/수정 모달 (관리자)
2. 출퇴근 버튼 + 기록 (작업자 홈에 추가)
3. 출퇴근 기록 조회 + 월별 근무 시간 집계 (관리자)
4. 휴가 신청 바텀시트 + 캘린더 (작업자)
5. 휴가 승인/반려 처리 (관리자)
6. 휴가 잔여 현황 테이블 (관리자)
7. 주간 근무 일정 타임라인 뷰 (관리자)

## 3단계: 농작업 관리

1. 작물 관리 CRUD (관리자)
2. 구역 관리 (관리자)
3. 작업 계획 + 배정 — 작물, 작업유형, 구역/열 선택 (관리자)
4. 작업 수령 + 시작/완료 — 타이머, 수량, 구역/열 입력 (작업자)
5. 작업 현황 보드 — 실시간 진행률 (관리자)
6. 생육 조사 입력 폼 + 이전값 비교 (작업자)
7. 이상 신고 — 사진, 유형, 코멘트 (작업자) + 확인 처리 (관리자)
8. 긴급 호출 (작업자 → 관리자)
9. 공지사항 작성 (관리자) + 열람 (작업자)

## 4단계: 통계 + 대시보드

1. 관리자 대시보드 — 출근 현황, 작업 진행, 알림 카드
2. 작업 성과 분석 — 작업자별/작물별/작업유형별 차트 (Recharts)
3. 근무 시간 통계
4. 수확량 추이 차트
5. 일일 보고서 생성

## 5단계: Supabase 연결

1. Supabase 프로젝트 생성
2. docs/db-schema.md 기반 테이블 마이그레이션
3. Zustand 목업 → Supabase 쿼리 전환 (hooks/ 수정)
4. Supabase Auth 연동 (PIN 로그인 커스텀)
5. Supabase Storage (사진 업로드)
6. Realtime 구독 (작업 현황 실시간)
7. RLS 정책 설정

## 6단계: 재배 AI 연동 (향후)

1. lib/ai-bridge.js 구현
2. 작업 완료 시 자동 전송 로직
3. AI 제안 수신 + 관리자 대시보드 표시
4. 예측 vs 실적 비교 차트
