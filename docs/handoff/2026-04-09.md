### 전체 Phase 플랜
- **Phase 1**: 사이드바 IA 재구성 + 스텁 페이지 추가 (✅ 완료)
- **Phase 2**: 근태 기능 확장 (대리 입력, 연장근무 통합 표시, 일괄 승인)

- **Phase 4**: 포장 관리 (신규 테이블 packaging_customers + packaging_tasks)
- **Phase 5**: 지점별 성과 분석 (관리팀 전용 뷰)

**모바일 하단 네비**: core 4탭 + 더보기 드로어 구조 유지, 스텁 5개를 더보기에 추가

### Phase 1에서 발견된 이슈 (Phase 2+로 이관)

**이슈 1: 역할별 사이드바 구성이 조사 전엔 불명확했음**
- 초기엔 "master는 hr_admin 사이드바와 동일"이라 추정했으나 실제로는 별개 사이드바 구성 필요했음
- 조사 단계(Claude Code가 수행)를 거쳐 정확한 구조 파악 후 설계 진행
- 교훈: 다음 Phase에서도 실제 코드 조사 먼저

**이슈 2: master가 farm_admin 페이지에 진입 시 currentUser.branch=NULL 처리 불명확**
- master 사이드바에 생산 관리 카테고리를 넣었지만 farm_admin 전용 페이지들이 branch 필터를 전제로 설계된 상태
- TaskPlanPage, TaskBoardPage, CropZonePage, GrowthSurveyAdminPage, IssueCallPage, StatsPage, WorkStatsPage, DailyReportPage 각각의 동작 확인 필요
- Phase 1에선 메뉴 노출만 하고 실제 페이지 동작 보장은 추후 대응

**이슈 3: AttendancePage vs AttendanceRecordsPage 관계 불명확**
- PC "근무 관리"와 모바일 core 탭 "출퇴근 기록"이 별개 페이지 컴포넌트
- 실제 용도 차이 불명확, Phase 2에서 코드 확인 후 통합 여부 판단

**이슈 4: LeaveApprovalRoute의 역할별 분기**
- /admin/leave-approval 경로가 역할에 따라 다른 컴포넌트 렌더 (farm_admin → LeaveApprovalPage, 나머지 → LeaveStatusPage)
- Phase 2에서 근태 기능 확장 시 이 구조 유지 or 단순화 판단

### 다음 세션 우선순위

**Phase 2 작업 상세**:

1. **attendance 테이블 컬럼 추가**:

2. **대리 입력 UI**:
