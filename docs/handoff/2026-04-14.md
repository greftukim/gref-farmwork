같은 세션에서 LLM 호출 실패 → 사용자 재전송 패턴 시 turn_index가 0, 2, 4, 6 등 짝수만 누적되되 비연속 가능. (assistant turn은 기록 안 되니 1, 3, 5는 비어있음)

LLM 컨텍스트는 messages 배열로 전달되니 영향 없음 (서버는 turn_index 검증 안 함). 다만 chat_logs 분석 시 빈 구간 발생.

### 5.4 Anthropic 크레딧 결제 이슈 (운영 메모)

세션 14 단위 5-C 재테스트 직전 시점에 Anthropic API 크레딧 부족 발생. 원인:
- 누적 사용으로 잔고 소진

조치:

- 충전 후 약 5~10분 지연 후 정상 반영
- 챗봇 재테스트 PASS 확인

향후 운영 권장:

- 정기 크레딧 잔고 모니터링 (BACKLOG CHATBOT-CREDIT-MONITOR-001 검토)

**추가**: 현재 Anthropic 결제는 태우 개인 카드 등록 상태. GREF 회사 계정 결제 전환 검토 필요 (회계 분리, 비용 추적, 퇴직 시 인수인계 등 이유). 별도 BACKLOG 등록 후보: CHATBOT-BILLING-CORPORATE-001.

## 7. 챗봇 운영 가이드 초안 (부산LAB 사용자 안내용)

> ⚠️ 이 절은 임시. H-4 또는 H-5 마감 후 별도 docs/operations/CHATBOT_USER_GUIDE.md로 분리 권장. 현 시점에는 base 자료로 HANDOVER에 보관.

### 7.1 노출 조건

farm_admin / hr_admin / master 권한 사용자만 우하단 FAB(💬) 표시.

### 7.2 사용 방법

- FAB 클릭 → 챗봇 패널 열림
- 환영 메시지 + 예시 질문 3개 표시
- 예시 질문 클릭 → 즉시 전송
- 직접 입력 → Enter 전송 (Shift+Enter 줄바꿈)
- 한국어 입력 중 Enter는 글자 확정 (자동 전송 안 됨)
- 최대 500자

### 7.3 응답 종류

- 운영 데이터 조회 (출근, 휴가, TBM, 대기 승인, 사용자 목록)
- 피드백 제출 (버그 신고, 개선 의견)
- 일반 대화 (제한적)

### 7.4 에러 발생 시

"일시적인 오류입니다" 또는 다른 에러 메시지 표시 시:
- 잠시 후 다시 시도
- 지속 발생 시 관리자 문의 (master 권한)

**관리자(태우) 안내**: Anthropic API 결제 이슈 시 크레딧 충전 후 5~10분 지연 반영. 사용자에게는 "일시적인 오류" 메시지로 노출되니 충전 후 잠시 후 재시도 안내.

### 7.5 새 대화

대화 1개 이상 후 새 대화(🔄) 버튼 활성. 클릭 시 confirm → 현재 세션 리셋.
