# 세션 보고 양식 체크리스트 (7항목)

세션 종료 시 이 양식을 복붙하여 순서대로 작성한다.  
각 항목을 생략하지 말 것. 없는 항목은 "없음" 으로 명시.

---

```
## 세션 [번호] 결과

### 1. 변경 파일
- 파일명: 변경 내용 한 줄
- ...
- 없음

### 2. 신규 교훈
- 교훈 [번호]: 제목
- ...
- 없음 (기존 교훈 재확인만)

### 3. BACKLOG 업데이트
- resolved: ID (사유)
- 신규: ID (등록 사유)
- 운영 후 트랙 이전: ID (사유)
- 없음

### 4. Task 0 分기 결과
- Task 0-1 ([항목]): 케이스 X/Y/Z/Z'/A — [한 줄 설명]
- Task 0-2 ([항목]): 케이스 X/Y/Z/Z'/A — [한 줄 설명]
- ...
- Task 0 없음 (단순 세션)

### 5. 트랙별 결과
- Track A ([제목]): [결과] — [한 줄 설명]
- Track B-1 ([제목]): [결과] — [한 줄 설명]
- ...
- 없음

### 6. Playwright
- PASS [n] / FAIL [n] / WARN [n]
- 이전 대비: +[n] / -[n] — [사유: 케이스 A/B/C]
- 재실행 여부: 없음 / [사유로 재실행]

### 7. 사용자 점검 권장 항목
- [URL 또는 기능]: [점검 요청 사유]
- ...
- 없음 (자동 검증 충분)

### GO/HOLD 판정
- 판정: GO / HOLD
- 사유: [한 줄]
- 다음 세션 후보: [제목 또는 "운영 진입"]
```

## 케이스 코드 정의

| 코드 | 의미 |
|------|------|
| X | 이미 구현됨 — 작업 불필요 |
| Y | 일부 구현 — 추가 작업 필요 |
| Z | 미구현 — 본 세션 구현 예정 |
| Z' | 미구현 + 외부 의존(DB 마이그레이션 등) — 보류 |
| A | 정상 상태 확인 — 조치 불필요 |
| B | 문제 발견 — 즉시 수정 |
| C | 누락/비활성 — 복원 필요 |

## Playwright 감소 케이스 코드

| 코드 | 의미 | 대응 |
|------|------|------|
| A | 정상 정리 — 이전 세션 검증 항목 제거, 신규 세션 검증 항목 추가 | 보고만 |
| B | 라벨/텍스트 미정정 — assertion 텍스트가 UI 변경과 불일치 | 즉시 정정 + 재실행 |
| C | 어서션 비활성화 — 신규 스크립트에서 이전 어서션 누락 | 복원 + 재실행 |
